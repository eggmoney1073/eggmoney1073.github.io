Monster World

오픈월드 몬스터 수집형 RPG를 목표로 제작한 프로젝트입니다. 플레이어 전투, 몬스터 포획과 소환, 상태 기반 AI, 데이터 테이블 구조, Shader Graph 활용까지 폭넓게 경험한 작업입니다.

프로젝트 개요

전투를 통해 몬스터를 사냥하고, 포획한 몬스터를 저장 및 소환하여 함께 전투할 수 있는 구조의 프로젝트입니다. 단순 전투 구현을 넘어서, 적과 아군이 같은 몬스터 오브젝트를 공유할 수 있도록 재사용성을 고려해 설계했습니다.

핵심 목표

  • 몬스터와 스킬 오브젝트를 재사용 가능한 구조로 제작
  • Custom State Machine으로 AI와 스킬 흐름을 일관되게 관리
  • JSON 기반 데이터 로딩으로 수치와 테이블을 코드에서 분리
  • Object Pooling과 Shader Graph로 성능과 표현을 동시에 개선

기술 포인트

Custom State Machine

상태의 Enter, Update, Exit를 Dictionary에 저장하는 방식으로 범용 상태머신을 만들고, 몬스터와 스킬 모두에 재사용했습니다.

Monster AI

적 몬스터는 순찰, 발견, 전투, 복귀 흐름을 따르며, 아군 몬스터는 플레이어와 전투 중인 적 유무를 기준으로 행동을 변경합니다.

Skill 시스템

Init, Charge, Attack, Finish의 4단계 상태 구조로 스킬을 만들고, SkillManager와 풀링 구조를 통해 재사용 가능한 형태로 구성했습니다.

데이터 기반 구조

  • MonsterLevelTable, MonsterTable, PlayerLevelTable, SkillTable 등 역할별 테이블 분리
  • Excel 데이터를 JSON으로 변환하는 별도 프로그램 제작
  • 레벨 수치, 몬스터 스탯, 스킬 정보를 외부 파일로 관리

최적화와 표현

  • Monster, Skill, Projectile, Effect에 Object Pooling 적용
  • 런타임 생성과 삭제를 줄여 성능 부담 완화
  • Shader Graph로 Toon Shader 직접 제작
  • Dithering 효과로 시야 가림 문제를 개선

성과

  • 복잡한 게임 로직을 모듈화된 C# 구조로 설계하고 구현하는 경험 축적
  • 데이터 기반 개발 흐름을 직접 구축하며 수치 조정과 테이블 관리 경험 확보
  • Object Pooling과 Shader Graph를 통해 성능과 시각 표현을 함께 다룬 경험 획득

배운 점

  • 프로젝트 초기에 시스템 구조를 명확히 설계하는 것이 유지보수성과 개발 효율에 큰 영향을 준다는 점
  • 모듈화와 재사용성이 가독성, 확장성, 생산성을 동시에 끌어올린다는 점
  • 핵심 데이터를 코드에서 분리하는 방식이 장기적으로 훨씬 유연하다는 점
