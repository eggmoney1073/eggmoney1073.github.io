Monster World

오픈월드 몬스터 수집형 RPG를 목표로 제작한 프로젝트입니다. 플레이어 전투와 몬스터 포획, 아군 몬스터 운용, 상태 기반 AI, 스킬 시스템, 데이터 테이블 구조, Shader Graph 활용까지 폭넓게 경험한 작업입니다.

프로젝트 개요

전투를 통해 몬스터를 사냥하고, 포획한 몬스터를 저장 및 소환하여 함께 전투할 수 있는 구조의 프로젝트입니다. 단순한 전투 구현을 넘어서, 적과 아군이 같은 몬스터 오브젝트를 공유할 수 있도록 재사용성을 고려해 설계했습니다.

  • 몬스터 사냥과 포획을 모두 성장 루프에 연결
  • 도감, 저장 슬롯, 재소환 등 운용 요소까지 포함
  • 아군과 적군 몬스터가 동일한 기본 구조를 공유하도록 설계

핵심 목표

  • 몬스터와 스킬 오브젝트를 재사용 가능한 구조로 제작
  • Custom State Machine을 이용해 AI와 스킬 흐름을 일관되게 관리
  • JSON 기반 데이터 로딩으로 수치와 테이블을 코드에서 분리
  • Object Pooling과 Shader Graph를 통해 성능과 표현을 동시에 개선

기술 포인트

Custom State Machine

상태의 Enter, Update, Exit를 Dictionary에 저장하는 방식으로 범용 상태머신을 만들고, 몬스터와 스킬 모두에 재사용했습니다. enum과 delegate를 결합해 여러 클래스에서 동일한 패턴으로 사용할 수 있게 구성했습니다.

Monster AI

적 몬스터는 순찰, 발견, 전투, 복귀 흐름을 따르며, 아군 몬스터는 플레이어와 전투 중인 적의 유무를 기준으로 행동을 변경합니다. 같은 몬스터 오브젝트가 아군과 적군 역할을 모두 수행하도록 설계했습니다.

Skill 시스템

스킬은 Init, Charge, Attack, Finish의 4단계 상태로 동작하며, SkillManager를 통해 요청되고 풀링된 오브젝트로 재사용됩니다. Target과 Caster를 인자로 받아 아군과 적군 모두 같은 구조로 사용할 수 있게 만들었습니다.

데이터 기반 구조

테이블 분리
MonsterLevelTable, MonsterTable, PlayerLevelTable, PlayerWeaponTable, SkillTable 등 역할별 테이블로 데이터 분리
Excel → JSON 변환
협업과 데이터 수정 편의성을 고려해 Excel 데이터를 JSON으로 변환하는 별도 프로그램 제작
코드와 수치 분리
레벨 수치, 몬스터 스탯, 스킬 정보 같은 핵심 데이터를 외부 파일로 관리해 수정 효율 확보

최적화와 렌더링

  • Monster, Skill, Projectile, Effect에 Object Pooling 적용
  • 실시간 생성/삭제를 줄여 런타임 부담 감소
  • Shader Graph로 Toon Shader를 직접 제작해 캐주얼한 분위기 표현
  • Dithering 효과를 추가해 플레이어를 가리는 오브젝트 표현 개선

구현한 주요 기능

  • New Game / Load Game / 세이브 슬롯 선택 구조
  • 전투를 통한 경험치 획득 및 레벨업 성장
  • 몬스터 포획, 저장, 소환 시스템
  • 도감 오픈 및 전체 포획 시 클리어 처리
  • NavMeshAgent 기반 이동과 장애물 회피

내가 강조하고 싶은 부분

  • 단순히 기능을 구현한 것이 아니라, 아군과 적군이 같은 구조를 공유하도록 재사용성까지 고려한 점
  • 상태머신, 스킬, 풀링, 데이터 테이블을 각각 따로 구현하는 데서 끝내지 않고 시스템적으로 연결한 점
  • 게임플레이 로직, 데이터 구조, 렌더링 표현을 한 프로젝트 안에서 모두 경험한 점

성과

  • 복잡한 게임 로직을 모듈화된 C# 스크립트로 설계하고 구현하는 경험 축적
  • 데이터 기반 개발 흐름을 직접 구축하며 수치 조정과 테이블 관리 경험 확보
  • Object Pooling과 Shader Graph를 통해 성능과 시각 표현을 함께 다룬 경험 획득

배운 점

  • 프로젝트 초기에 시스템 구조를 명확히 설계하는 것이 유지보수성과 개발 효율에 큰 영향을 준다는 점
  • 모듈화와 재사용성이 가독성, 확장성, 생산성을 동시에 끌어올린다는 점
  • 핵심 데이터를 코드에서 분리하는 데이터 기반 개발 방식이 장기적으로 훨씬 유연하다는 점
