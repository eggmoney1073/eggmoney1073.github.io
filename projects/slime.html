Slime Defense

Unity ECS(DOTS) 기반으로 제작 중인 모바일 디펜스 프로젝트입니다. 고정된 플레이어를 중심으로 몰려오는 적을 상대하며, 전투와 성장 선택지를 반복하는 짧고 밀도 높은 루프를 목표로 설계했습니다.

GitHub 메인으로
장르 모바일 디펜스 + 성장형
엔진 Unity
핵심 기술 ECS, Addressables, UI
개발 형태 1인 개발

프로젝트 개요

플레이어는 화면 중앙에 고정된 상태로 터치 입력을 통해 조준 방향을 제어하고, 적을 처치하며 경험치를 획득합니다. 레벨업 시 3지선다 선택지를 통해 무기를 강화하고, 이를 반복하며 한 판의 성장을 완성하는 구조입니다.

핵심 루프

전투
적 등장, 조준, 공격, 처치의 반복
성장
경험치 획득 후 레벨업 선택지 제공
강화
공격력, 발사 속도, 관통, 투사체 수 강화

기술 포인트

ECS 시스템 그룹 설계

관리, 생성, 이동, 판정, 계산 단계를 그룹으로 나누고 시스템 순서를 명시해, 프레임 처리 흐름이 명확하게 보이도록 구성했습니다.

레벨업 선택지 데이터화

레어도, 강화 타입, 수치로 선택지를 분리해 설계하고, 이후 수치 조정과 신규 선택지 추가가 쉬운 구조를 만들었습니다.

Addressables 활용

UI와 리소스 참조를 직접 경로에 의존하지 않도록 정리하며, 점진적으로 관리 가능한 리소스 구조로 개선하고 있습니다.

내가 집중한 부분

  • 시스템 흐름이 꼬이지 않도록 업데이트 순서와 책임을 분리한 점
  • 모바일 입력과 UI 피드백이 즉각적으로 느껴지도록 조정한 점
  • 초반과 후반 성장 체감 차이를 수치 밸런스로 다듬은 점
  • 데이터 기반 구조를 먼저 잡아 이후 확장이 쉬운 형태로 만든 점

배운 점

  • 기능 추가보다 시스템 흐름을 먼저 정리하는 편이 훨씬 효율적이라는 점
  • 데이터 분리가 되어 있어야 밸런스 조정과 유지보수가 쉬워진다는 점
  • 모바일 게임에서는 조작 반응성과 UI 응답성이 플레이 체감에 직접 영향을 준다는 점
