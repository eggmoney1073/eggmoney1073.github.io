Slime Defense

Unity ECS 기반으로 제작한 모바일 디펜스 프로젝트입니다. 이 프로젝트에서는 단순히 적과 투사체를 움직이는 수준이 아니라, 시스템 실행 순서를 그룹으로 분리하고, Grid 기반 충돌 판정을 통해 ECS 구조에 맞는 전투 루프를 만들었습니다.

Overview

프로젝트 개요

무엇을 만들었는가

플레이어는 중앙에서 조준 방향만 제어하고, 적은 경로를 따라 이동하며, 투사체는 ECS 시스템에서 생성되고 이동합니다. 충돌 판정과 대미지 누적, 삭제 처리까지 모두 시스템 단위로 분리해 전투 루프를 구성했습니다.

왜 중요한가

이 구조는 기능이 많아질수록 강해집니다. 스폰, 이동, 판정, 계산, 삭제가 섞이지 않기 때문에 디버깅이 쉬워지고, 새로운 무기나 적 타입을 추가할 때도 영향 범위를 줄일 수 있습니다.

Architecture

핵심 구조 분석

1. 시스템 그룹으로 전투 흐름을 분리

이 프로젝트의 가장 핵심은 업데이트 순서를 명확히 고정한 점입니다. 관리, 생성, 이동, 판정, 계산, 삭제를 별도 그룹으로 두어 한 프레임 안에서도 어떤 작업이 먼저 실행되는지 구조적으로 보장했습니다.

2. Grid 기반 충돌 판정

모든 적을 매 프레임 전수 조사하지 않고, 적을 Grid 해시에 넣은 뒤 투사체 주변 셀만 검사하는 방식으로 충돌 대상을 좁혔습니다. ECS에서 많은 개체를 다룰 때 중요한 구조입니다.

3. 브리지 구조로 MonoBehaviour와 ECS 연결

조준 방향 같은 입력값은 MonoBehaviour 쪽에서 받아도, 실제 발사는 ECS 시스템에서 이 값을 읽어 처리합니다. UI와 입력은 익숙한 방식으로 유지하면서, 핵심 전투 로직만 ECS에 실은 구조입니다.

4. 단계별 후처리 분리

충돌에서 바로 삭제하지 않고, 충돌 기록 → 대미지 누적 → 생존 판정 → 삭제로 분리했습니다. 이 방식은 ECS에서 병렬성과 안정성을 유지하는 데 유리합니다.

Flow

프레임 처리 흐름

Manage

TimeScaleData를 갱신해 전체 시스템이 사용할 공통 시간값을 먼저 정리합니다.

Spawn

적을 경로 시작점에 생성하고, 무기는 조준 방향을 기준으로 투사체를 발사합니다.

Move

적은 웨이포인트를 따라 이동하고, 투사체는 방향 벡터를 따라 직선 이동합니다.

Judgement

Grid를 구축한 뒤 투사체 주변 셀만 검사해서 충돌 여부를 판단합니다.

Calculate

적이 받은 대미지 버퍼를 합산해 체력을 깎고, 관통 수나 생존 여부를 계산합니다.

Destroy

죽은 엔티티나 수명이 끝난 엔티티를 마지막 단계에서 정리합니다.

System Flow

시스템 그룹 순서 고정

전투 흐름의 뼈대다. 어떤 시스템이 어느 순서로 실행되는지 코드 레벨에서 고정해, 프레임 처리 순서를 예측 가능하게 만든다. 

[UpdateInGroup(typeof(SimulationSystemGroup))]

[UpdateInGroup(typeof(SlimeDefenseSystemGroup))]
[UpdateAfter(typeof(ManageSystemGroup))]

Spawn

경로 기반 적 생성

적 생성 시 웨이포인트 시작 위치를 가져오고, Path 참조와 체력 데이터를 함께 부여한다. 단순 Instantiate가 아니라 전투에 필요한 상태를 한 번에 세팅한다. 

Entity enemy = ecb.Instantiate(spawner.ValueRO.enemyPrefab);

float3 spawnPos = waypoints[0].nodePosition;
spawnPos.z += _spawnIndex * 0.001f;

ecb.SetComponent(enemy, LocalTransform.FromPosition(spawnPos));

ecb.AddComponent(enemy, new EntityIndex
{
    index = _spawnIndex
});

ecb.AddComponent(enemy, new ECS_PathReference
{
    path = pathEntity
});

ecb.AddComponent(enemy, new EnemyHealth
{
    health = 100f + (_spawnIndex * 100f * spawner.ValueRO.enemyHealthSale)
});
Bridge + Fire

입력과 ECS 발사를 연결

조준 방향은 브리지에서 저장하고, 무기 발사 시스템은 그 값을 읽어 투사체를 생성한다. UI 입력과 ECS 전투 로직을 깔끔하게 분리한 구조다. 

float3 dir = AimDirectionBridge.AimDirection;
float angle = math.atan2(dir.y, dir.x);

Entity projectileEntity = ecb.Instantiate(projectile.ValueRO.projectileEntity);

ecb.SetComponent(
    projectileEntity,
    LocalTransform.FromPositionRotationScale(
        new float3(0, 0, _spawnIndex * -0.001f),
        quaternion.RotateZ(angle),
        1f
    )
);

ecb.AddComponent(projectileEntity, new Direction
{
    moveDirection = dir
});
Grid Collision

Grid 구축 후 주변 셀만 검사

충돌 판정의 성능 포인트다. 적을 Grid에 넣고, 투사체는 중심 셀 기준 3x3 범위만 조회해서 충돌 후보를 찾는다. 

GridBuild_Job gridJob = new GridBuild_Job
{
    cellSize = cellSize,
    gridWidth = gridWidth,
    gridMap = _gridMap.AsParallelWriter()
};

Collision_Job collisionJob = new Collision_Job
{
    _cellSize = cellSize,
    _gridWidth = gridWidth,
    _gridMap = _gridMap,
    _hitBufferLookup = SystemAPI.GetBufferLookup<HitEnemyBufferElement>(false),
    _transformLookup = SystemAPI.GetComponentLookup<LocalTransform>(true),
    _radiusLookup = SystemAPI.GetComponentLookup<Collision>(true),
    _damagedBufferLookup = SystemAPI.GetBufferLookup<Damaged>(false)
};
Damage

충돌과 체력 계산 분리

충돌 시 바로 체력을 깎지 않고 Damaged 버퍼에 기록한 뒤, 다음 계산 단계에서 한 번에 처리한다. 단계 분리 덕분에 병렬 작업 구조를 유지하기 좋다. 

public void Execute(
    Entity enemy,
    ref DynamicBuffer<Damaged> damage,
    ref EnemyHealth health,
    EnabledRefRW<LiveTag> liveTag)
{
    for (int i = 0; i < damage.Length; i++)
    {
        health.health -= damage[i].damage;
    }

    damage.Clear();
}
Destroy

마지막 단계에서 엔티티 정리

삭제를 별도 그룹으로 미뤄서 프레임 중간의 참조 꼬임을 줄였다. ECS에서 안전한 후처리 흐름을 만들기 위한 선택이다. 

EntityCommandBuffer entityCommandBuffer =
    new EntityCommandBuffer(Allocator.TempJob);

EntityDestroy_Job destroyJob = new EntityDestroy_Job
{
    ecbWriter = entityCommandBuffer.AsParallelWriter()
};

state.Dependency = destroyJob.ScheduleParallel(state.Dependency);
state.Dependency.Complete();

entityCommandBuffer.Playback(state.EntityManager);
entityCommandBuffer.Dispose();

What I Learned

이 프로젝트에서 보여줄 수 있는 역량

ECS 구조 설계 능력

단순히 ECS 문법을 사용한 것이 아니라, 실제 전투 루프를 시스템 그룹 단위로 나눠 설계했습니다. 이 점이 프로젝트의 가장 큰 기술 포인트입니다.

성능을 고려한 판정 구조

Grid 해시를 도입해 충돌 후보를 줄였고, 충돌과 계산을 분리해 다수 개체 처리에 맞는 구조를 잡았습니다.
