Slime Defense

Unity ECS(DOTS) 기반으로 제작 중인 모바일 디펜스 프로젝트입니다. 고정된 플레이어를 중심으로 몰려오는 적을 상대하며, 전투와 성장 선택지를 반복하는 짧고 밀도 높은 플레이 루프를 목표로 설계했습니다.

프로젝트 개요

플레이어는 화면 중앙에 고정된 상태로, 터치 입력으로 조준 방향을 제어하며 적을 처치합니다. 전투를 통해 경험치를 획득하고, 레벨업 시 3지선다 선택지를 통해 무기를 강화하는 루프를 반복합니다.

  • 한 판 플레이 타임이 짧고 반복이 빠른 모바일 루프 지향
  • 전투 → 경험치 획득 → 레벨업 → 강화 → 재전투 흐름 중심
  • 터치 조작 기반 조준과 피드백이 핵심인 구조

내가 집중한 부분

  • 시스템 업데이트 순서를 그룹 단위로 정리해 흐름을 명확히 구성
  • 레벨업 선택지와 강화 수치를 데이터 형태로 관리할 수 있게 구조화
  • 입력, UI, 전투 판정이 서로 꼬이지 않도록 책임을 분리
  • 모바일 환경에서 간단하고 즉각적인 조작감을 유지하는 방향으로 설계

기술 포인트

ECS 기반 시스템 그룹 구성

관리, 생성, 이동, 판정, 계산 단계를 그룹으로 나누고 각 시스템의 순서를 명시해, 프레임마다 어떤 처리가 어느 단계에서 일어나는지 명확히 보이도록 구성했습니다.

레벨업 선택지 데이터 설계

무기 강화 선택지를 레어도, 강화 타입, 수치로 분리해 데이터 중심으로 관리하고, 이후 선택지 추가나 수치 조정이 쉽게 이어지도록 구조를 잡았습니다.

Addressables 기반 리소스 관리

UI 프리팹과 리소스를 직접 경로 문자열에 의존하지 않는 방향으로 정리하며, 로딩 흐름과 참조 방식을 점차 관리 가능한 구조로 바꾸고 있습니다.

게임 루프 설계

1. 전투 시작
적이 경로를 따라 등장하고 플레이어는 터치 조작으로 조준을 제어합니다.
2. 처치 및 경험치 획득
적 처치 결과가 경험치로 환산되고 플레이어 성장 조건을 채웁니다.
3. 레벨업 3지선다
강화 선택지를 제시하고, 플레이어는 플레이 스타일에 맞게 성장 방향을 고릅니다.
4. 강화 반영
공격력, 발사 속도, 관통, 투사체 수 등 전투 성능이 즉시 반영됩니다.

입력과 UI

  • 모바일 환경을 고려한 터치 조이스틱 및 조준 방향 제어 방식 구현
  • 경험치, 타이머, 사격 방향 표시 UI를 전투 흐름에 맞춰 배치
  • 터치가 유효한 영역을 제한하고, 범위를 벗어날 때 즉시 종료되는 입력 처리 고민
  • 실제 플레이 감각을 기준으로 UI 피드백과 조작 응답성을 조정

문제 해결 경험

  • ECS 구조에서 시스템 순서가 꼬일 때 그룹 책임을 다시 나누는 방식으로 흐름을 재정리
  • Addressables 사용 시 하드코딩 경로 의존도를 줄일 방법을 고민하고 구조를 개선
  • 레벨업 선택지 설계에서 초기 구간과 후반 구간의 성장 체감 차이를 수치 밸런스로 조정
  • 모바일 입력 처리에서 범위를 벗어난 터치 종료 같은 세부 UX까지 직접 다듬음

배운 점

  • 기능을 빠르게 추가하는 것보다, 시스템 흐름을 먼저 정리하는 편이 전체 개발 효율이 높다는 점
  • 데이터 기반 구조를 먼저 잡아두면 이후 밸런스 조정과 기능 확장이 훨씬 쉬워진다는 점
  • 모바일 게임에서는 작은 조작 지연이나 UI 응답 차이도 플레이 체감에 크게 영향을 준다는 점
  • 구조, 입력, UI, 성장 루프는 따로가 아니라 함께 맞물려야 플레이가 자연스럽게 나온다는 점
